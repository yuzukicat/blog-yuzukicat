### 5 因为公司发生的事故或事件被追究责任并被要求事后对应

#### 心理负荷强度 中

#### 被要求对 FoodPlatingAI 项目的失败负全部责任并被要求事后对应

这主要发生在2022年年末~2023年2月底。

我作为后端工程师按时完成了自己的工作。但老板要求我对项目未能如期完成负全部责任。老板批评我缺乏责任心，说我作为全职员工和作为从最开始参与这个项目员工，应该对项目失败负有全部责任。

我在事后或者说是该项目被废弃之前，仍然被要求以PM的角色，召集兼职前端Ramon和新入职的UI/UX设计师Maneesha一起开会，我被要求问他们能否在一周之内完成并改善前端设计，以挽救项目的失败。

### 6 在自己参与的工作中，发生重大损失并被要求事后对应

#### 心理负荷强度 中

#### 我被要求做 Dall.E 2 的项目落地，在使用 AWS Machine Learning 相关服务的过程中，发生 30 多万日元的巨额损失，在损失发生后，我被要求与 AWS 客服进行邮件 (英语沟通) 以减小或者挽回损失。

Eyes, Japan Co,Ltd. 的老板山寺纯想做 Dall.E 2 的项目落地，我被要求和多位兼职员工一起使用 AWS Machine Learning 相关服务进行研发。我们使用 AWS P2 实例作为 Jupyter Nootbook 的计算资源。

一周内的使用费用共计30多万日元，远远超过了这家创业10年仍然在创业的小型创业公司的经济承受能力。

其实对于 AWS 的费用是有邮件提醒的，我也看到了，但在此期间没有人提出花费有问题，直到月底收到账单。

我被要求对此“巨额损失”负有全部责任，并被要求和 AWS 客服进行邮件 (英语沟通) 以减小或者挽回损失。

#### 证据

有 AWS 的工单和邮件作为证据。

相关参与者作为人证(多位兼职员工)。
